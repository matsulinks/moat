# AI仲裁エージェント仕様 v1.0

## 1. 目的

新しいソフトウェアをラズパイにデプロイした際に発生する
「セキュリティポリシーとの衝突」または「新ソフト自体の脆弱性」を検知し、
ユーザーへのレポートと、そのソフトを開発したAIへの報告・交渉を自動化する。

**解決する問題**:
- 新ソフトがセキュリティルールでブロックされて動かない
- 新ソフトに脆弱性があってもユーザーが気づかない
- 問題の原因と解決策を技術的に分析できない

## 2. アーキテクチャ

```
[新ソフトデプロイ]
      ↓
[衝突検知エンジン]─────────────────────────┐
  ・Falcoイベント監視                      │
  ・Layer 6 ツール拒否ログ監視             │
  ・起動失敗・クラッシュ検知               │
      ↓                                   │
[AI分析エンジン（OpenAI API）]             │
  ・問題の根本原因を分析                   │
  ・「セキュリティ側を緩める」vs          │
    「ソフト側を修正する」を判断           │
      ↓                                   │
[ユーザーレポート]                         │
  ・Telegram通知                           │
  ・問題の説明（平易な言葉）              │
  ・推奨アクション提示                    │
      ↓（ユーザー承認）                   │
[開発AI への報告]                          │
  ・GitHub Issue作成（自動）               │
  ・クリップボードコピー（手動）           │
  ・どちらかはユーザー設定で選択          │
      ↓                                   │
[解決策の適用]                             │
  ・セキュリティ例外を追加（人間承認必須）│
  ・ソフト側の修正を待つ                  │
└─────────────────────────────────────────┘
```

## 3. 衝突検知の対象

| 検知種別 | 具体例 | 重要度 |
|---|---|---|
| Layer 6 ツール拒否 | 新ソフトがexec/fs:writeを使おうとしてブロック | ★★★ |
| Falco CRITICAL | 新コンテナがshellを起動しようとした | ★★★★★ |
| 起動失敗 | cap_drop:ALLでPermission Denied | ★★★ |
| ネットワーク拒否 | 許可ドメイン外への通信試行 | ★★★ |
| 脆弱性パターン | 既知の危険なコード構造を検知 | ★★★★ |

## 4. AI分析エンジン

### 4.1 送信する情報（匿名化必須）

```json
{
  "event_type": "layer6_tool_denied",
  "tool_requested": "exec",
  "software_name": "新プロジェクト名",
  "software_repo": "https://github.com/xxx/yyy",
  "log_snippet": "Permission denied: exec not in allowlist",
  "security_layer": 6,
  "timestamp": "2026-03-03T19:00:00Z"
}
```

**送信しない情報**: IPアドレス、パスワード、トークン、個人情報、
ホスト名、ディレクトリ構造の詳細

### 4.2 AI分析プロンプトテンプレート

```
あなたはOpenClawのセキュリティアドバイザーです。
以下のセキュリティ衝突イベントを分析してください。

イベント: {event_type}
対象ソフト: {software_name}
詳細: {log_snippet}

以下を回答してください:
1. 問題の根本原因（技術的説明）
2. ユーザー向け説明（非技術者でも理解できる言葉で）
3. 推奨アクション: [A] ソフト側の修正 / [B] セキュリティ例外追加 / [C] 両方
4. 開発AIへの報告文（GitHub Issue形式）
5. リスクレベル: CRITICAL / HIGH / MEDIUM / LOW
```

### 4.3 利用するAPI

- **OpenAI API**（gpt-4o推奨）
- APIキーはInfisicalで管理（平文保存禁止）
- レート制限: 1イベントにつき1回のAPI呼び出し（連続呼び出し防止）
- タイムアウト: 30秒

## 5. 開発AIへの報告

### 5.1 GitHub Issue形式（自動作成モード）

```markdown
## 🔒 Security Conflict Report

**Generated by**: OpenClaw Security-System AI Mediation Agent  
**Software**: {software_name}  
**Date**: {date}  

### 問題の概要
{ai_generated_summary}

### 検知されたイベント
- Layer: {layer}
- Event: {event_type}  
- 詳細: {log_snippet}

### 推奨対応
{ai_recommended_fix}

### リスクレベル
{risk_level}

---
*このIssueはAIが自動生成しました。人間によるレビューを推奨します。*
```

### 5.2 クリップボードコピーモード

ユーザーが手動でペーストして使う。
フォーマットは上記Issue形式と同一。

### 5.3 ユーザー設定（config.yaml）

```yaml
ai_mediation:
  enabled: true
  report_mode: "clipboard"   # "github_issue" or "clipboard"
  github_token: "{{ INFISICAL_GITHUB_TOKEN }}"  # github_issueモード時のみ必要
  ai_api:
    provider: "openai"
    model: "gpt-4o"
    api_key: "{{ INFISICAL_OPENAI_API_KEY }}"
  anonymize: true             # 送信前に個人情報を除去（常にtrue推奨）
  approval_level: high_risk_only  # none / high_risk_only / all
  notification:
    telegram: true
```

## 6. メッセージの2層構造

セキュリティのエラー・警告はすべて以下の2層で表示する。

### Layer A: ユーザー向け（平易な言葉）
- 何が起きているかを1〜2文で説明
- 今すぐやることを箇条書きで1〜3個
- 技術用語・ログ・コードは含めない

### Layer B: AI向け詳細（技術情報）
- ログの生データ
- イベント種別・Layer番号・リスクレベル
- 推奨修正コード・コマンド
- MITRE ATT&CK分類

**表示例（Telegram通知）:**


ユーザーはLayer Aだけを見て判断できる。
詳細が必要なときだけLayer Bを展開する。
開発AIへの報告にはLayer Bのみを送信する。

## 6. 人間承認フロー

```
AI分析完了
    ↓
Telegramに通知:
  「新ソフトXXXとLayer 6が衝突しています。
   原因: exec権限が必要ですが許可されていません。
   推奨: ソフト側でexecを使わない実装に変更。
   開発AIに報告しますか？ [はい] [いいえ] [詳細]」
    ↓
[はい] → GitHub Issue作成 or クリップボード
[いいえ] → ログのみ記録
[詳細] → AI分析の全文を表示
```

**自動送信は絶対に行わない。必ず人間の承認を待つ。**

## 7. セキュリティ考慮点

| リスク | 対策 |
|---|---|
| AI Mediator自体が攻撃面になる | OpenAI APIへの通信はホワイトリスト必須 |
| 分析結果に悪意ある内容が混入 | AIレスポンスをサニタイズ、表示前にチェック |
| 機密情報がAPIに送信される | 送信前に匿名化処理（anonymize: true） |
| API呼び出しのなりすまし | APIキーはInfisical管理、通信はTLS必須 |
| 自動でIssueを大量作成される | レート制限 + 人間承認必須 |

## 8. 外部通信ホワイトリスト（本エージェント専用）

```yaml
ai_mediation_whitelist:
  - "api.openai.com:443"          # AI分析API
  - "api.github.com:443"          # Issue作成（github_issueモード時）
```

これ以外のエンドポイントへの通信は禁止。
